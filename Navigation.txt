Навигационные свойства
Это свойства (properties-{get;set;}) в классах-сущностях (Entity), которые
отвечают за связи с другими сущностями.

public class Manager
{
    public Guid IdMaindep { get; set; }  // свойство (данных)
    public Department MainDep { get; set; }  // навигационное свойство
}

Это позволяет упростить получения связанных данных - не отправлять отдельный запрос на поиск,
а воспользоваться неявными возможностями.

Шаги реализации:
1. Объявляем свойство в Entity:
     public Department MainDep { get; set; }
2. Настраиваем свойство в контексте данных (DataContext):
     - переопределяем override метод -> protected override void OnModelCreating(ModelBuilder modelBuilder) {}
       в этом методе происходит настройка самой БД - отношений между данными, их ограничение и уникальность,
       а также засевание (от англ. seed - зерно) - заполнение начальными данными.
3. После настройки отношения нужно сделать и применить миграцию.
